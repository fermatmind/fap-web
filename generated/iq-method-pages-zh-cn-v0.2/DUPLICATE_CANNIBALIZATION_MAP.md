# Duplicate / Cannibalization Map

Status: `draft_review_only`

## Purpose

This map defines the unique search intent for seven IQ method pages so they do not cannibalize one another or existing FermatMind IQ articles.

## Known duplicate-risk themes

- 在线 IQ 测试准不准
- IQ 结果怎么看
- 原始分与正确率解释
- 在线测试和专业测评区别
- 矩阵推理题型说明
- 为什么 V1 不提供正式认证
- IQ 数据和隐私边界
- 博士/专家背书的边界

Any older broad copy that uses unsupported claims must go through claim cleanup. Do not copy old wording that implies true IQ, external proof, formal ranking, certificate, or clinical use.

## Unique intent per page

| Page | Unique search intent | Avoid competing with |
|---|---|---|
| WHAT-IS-IQ-STYLE-TEST | Define IQ 风格推理测试 and explain its scope. | Score meaning, professional assessment comparison. |
| ONLINE-VS-PROFESSIONAL | Compare online practice/self-understanding vs professional assessment. | Generic "what is IQ-style" definition. |
| SCORE-MEANING-BOUNDARY | Explain raw score, correct rate, completion time. | Formal score/ranking pages or professional assessment pages. |
| MATRIX-REASONING-GUIDE | Explain matrix/visual/pattern task format. | Item-level answer/explanation pages, which are forbidden. |
| WHY-NOT-CERTIFICATION | Explain why V1 is non-certification. | Expert review disclosure and privacy page. |
| PRIVACY-AND-DATA-BOUNDARY | Explain data/public-private/answer-key boundaries. | General privacy policy page. |
| EXPERT-REVIEW-DISCLOSURE | Explain advisory review without overclaiming. | Certification page or official authority claims. |

## Existing FermatMind IQ content relationship

These pages should support the IQ test landing page and existing IQ articles. They should not become new money pages for "IQ test" itself. The IQ test page remains the owner for high-intent test-start queries.

## Cleanup reminders

- Do not use old copy that says or implies official external proof.
- Do not use old copy that suggests formal ranking or certificate output.
- Do not use old copy that exposes internal scale codes as user-facing positioning.
- Do not use article pages as result pages.
