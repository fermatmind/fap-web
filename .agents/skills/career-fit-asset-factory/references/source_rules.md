# Career Fit Source Rules

Prefer O*NET interests/work styles/work values and FermatMind test framework documentation. Treat MBTI and Big Five as supplementary style lenses.

Every source item must include source name, URL or citation handle, captured fact, source boundary, and whether the source is direct, aggregate, adjacent, or macro context.

For reader-facing Big Five bridge copy, the FermatMind source must be the
backend-published public projection for the same locale and authority identity.
Generated packages, imported primary drafts, working revisions, private result
payloads, and frontend fallback copy are not public sources.

Keep occupation evidence and personality interpretation evidence separate. A
Big Five source may support bounded work-style language; it does not prove that
an occupation is suitable or predict an employment outcome.
